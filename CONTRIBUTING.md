# Contributing

Thanks for improving `cybersecurity-playbook`.

## Contribution goals

This repo should stay:
- reusable
- markdown-first
- practical
- clearly scoped as a companion content library

## Good contributions

- better templates
- stronger ATT&CK / telemetry / triage examples
- clearer markdown standards
- useful sample detections or investigation notes
- better folder organization when it genuinely reduces confusion

## Avoid

- turning the repo into a pseudo-platform
- adding unsupported vendor claims
- committing tenant-specific, secret, or sensitive internal content
- filling templates with unrealistic placeholder output

## Workflow

1. make a focused branch
2. keep content changes scoped
3. preview markdown before opening a PR
4. explain what template or content pattern improved
